# Проектирование госуслуг (public-service-design)

Спроектировать логику предоставления государственной услуги онлайн с учетом требований федерального законодательства (ФЗ-210) и интеграции с ведомствами.

## Процесс
1. **Определи SLA предоставления услуги.** Сроки ответа ведомств по регламенту (например, не более 5 рабочих дней).
2. **Спроектируй межведомственное взаимодействие (СМЭВ).** Какие данные запрашиваются из баз МВД, Росреестра, ФНС автоматически без участия гражданина.
3. **Опиши омниканальный опыт.** Личный кабинет госуслуг, МФЦ (офлайн), email-уведомления.

## Формат вывода
```
## Дизайн госуслуги: [Название услуги]
- **Регуляторное SLA:** [N дней на обработку заявления].
- **Интеграция СМЭВ:** автоматический запрос данных из ЕГРН Росреестра при валидации права собственности.
```

## Метрики

### Универсальное правило метрики
Если вы предлагаете метрику, ответьте на 5 вопросов:
1. **Кто владеет этой метрикой?**
2. **Как часто её смотрим?**
3. **Какие события её считают?**
4. **Какой порог решения?**
5. **Как её можно испортить или накрутить?**
