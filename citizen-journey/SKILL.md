# Путь гражданина (citizen-journey)

Спроектировать путь гражданина при получении государственной услуги онлайн (например, Госуслуги, Мос.ру), выявить точки трения и доступность сервиса для всех категорий лиц.

## Процесс
1. **Опиши шаги Citizen Journey Map.** Поиск услуги -> Проверка прав -> Заполнение формы -> Прикрепление документов -> Статус заявления -> Результат.
2. **Оцени доступность (Accessibility).** Соответствие стандартам WCAG (для слабовидящих, пожилых, мобильных устройств).
3. **Выяви точки трения (Friction points).** Сбои авторизации по Госуслугам (ЕСИА), сложные юридические формулировки.

## Формат вывода
```
## Citizen Journey Map: [Название услуги]
- **Целевой гражданин:** [профиль, например: пенсионер, оформляющий льготу]
- **Точки отказа (Drop-off):** падение конверсии на шаге загрузки скана паспорта в формате PDF.
- **Рекомендации по доступности:** упростить форму, использовать автозаполнение данных из личного кабинета Госуслуг.
```

## Метрики

### Универсальное правило метрики
Если вы предлагаете метрику, ответьте на 5 вопросов:
1. **Кто владеет этой метрикой?**
2. **Как часто её смотрим?**
3. **Какие события её считают?**
4. **Какой порог решения?**
5. **Как её можно испортить или накрутить?**
