# Проектирование админ-панелей (admin-ux)

Спроектировать UX/UI админ-панели (back-office) для сотрудников поддержки, модераторов или операторов маркетплейса/банка.

## Процесс
1. **Фокусируйся на задаче (Task-oriented UX).** Админка должна быть быстрой и функциональной, красота на втором месте.
2. **Спроектируй защиту от дурака (Error Prevention).** Подтверждение критических действий (удаление, блокировка, смена тарифа), логирование изменений (Audit Trail).
3. **Опиши массовые операции (Bulk Actions).** Выделить 100 пользователей -> заблокировать одной кнопкой.

## Формат вывода
```
## UX Спецификация админ-панели: [Модуль]
- **Критическое действие (Удаление аккаунта):** требует ввода слова «УДАЛИТЬ» для подтверждения действия оператором.
- **Audit Trail:** логирование IP оператора, времени и старого/нового значения параметров в БД при любом изменении.
```

## Метрики

### Универсальное правило метрики
Если вы предлагаете метрику, ответьте на 5 вопросов:
1. **Кто владеет этой метрикой?**
2. **Как часто её смотрим?**
3. **Какие события её считают?**
4. **Какой порог решения?**
5. **Как её можно испортить или накрутить?**
