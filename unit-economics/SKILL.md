# Юнит-экономика финансовых продуктов (unit-economics)

Построить детальную модель юнит-экономики для финансового, кредитного или ипотечного продукта. Скилл помогает продакту рассчитать пожизненную ценность клиента (LTV) с учетом процентных доходов, стоимости фондирования (CoF), резервов на возможные потери по кредитам (LLP), транзакционных расходов, стоимости привлечения (CAC) и срока окупаемости (Payback Period).

## Процесс

1. **Определи юнит (единицу анализа).** (Обычно это один выданный кредит, одна активная карта или один привлеченный клиент).
2. **Собери структуру доходов (Revenue Drivers).**
   - *Interest Revenue (Процентный доход):* проценты по кредиту.
   - *Fee Revenue (Комиссионный доход):* комиссии за выдачу, страховки, SMS-информирование, эквайринг.
3. **Собери структуру расходов (Cost Drivers).**
   - **Cost of Funds (CoF / Стоимость фондирования):** стоимость денег для банка (например, ставка по депозитам или ключевая ставка ЦБ РФ).
   - **Expected Loss / LLP (Резьв на потери):** стоимость кредитного риска. Рассчитывается как `PD (вероятность дефолта) × LGD (убыток при дефолте)`.
   - *Operating Costs (Транзакционные расходы):* ФОТ андеррайтеров, скоринг-запросы в БКИ, проведение расчетов.
4. **Рассчитай окупаемость.**
   - **LTV:** суммарный маржинальный доход за весь срок жизни продукта (с учетом досрочного погашения).
   - **CAC:** затраты на маркетинг и продажи, деленные на количество выданных продуктов.
   - **LTV / CAC:** соотношение должно быть > 3 для стабильной бизнес-модели.
5. **Сохрани вывод** в текущей рабочей директории как `unit-economics-[название-продукта].md`.

## Формат вывода

```
## Юнит-экономика кредитного продукта: [Название продукта]
- **Базовый юнит:** Один выданный потребительский кредит (средний размер [X] руб., срок [Y] месяцев).

### 1. Доходы на один юнит (Revenue per Unit)
- **Процентный доход (Interest Revenue):** [X] руб. за весь срок (ставка [Y%] годовых).
- **Комиссионный доход (Fees & Cross-sell):** [Z] руб. (включая страхование жизни, комиссии за оформление).
- **Итого доход на один юнит (Lifetime Revenue):** [R] руб.

### 2. Расходы на один юнит (Cost of Goods Sold / COGS)
- **Стоимость фондирования (Cost of Funds - CoF):** [C1] руб. (исходя из ставки привлечения депозитов банка в [W%] годовых).
- **Кредитный риск (Expected Loss / LLP):** [C2] руб. (на основе исторической вероятности дефолта PD = [A%] и тяжести потерь LGD = [B%]).
- **Транзакционные косты (Operations Cost):** [C3] руб. (запросы в БКИ, скоринг, ФОТ верификации, СБП комиссии).
- **Итого расходы на один юнит (Lifetime COGS):** [Total Costs] руб.

### 3. Маржинальность и Сходимость экономики
- **Lifetime Value (LTV):** [Revenue - COGS] = [LTV] руб.
- **Cost of Acquisition (CAC):** [CAC] руб. (маркетинг + агентская комиссия партнерам/риелторам за лид).
- **Чистая прибыль с юнита (Contribution Margin):** [LTV - CAC] = [Margin] руб.
- **LTV / CAC Ratio:** [LTV / CAC] (Оценка здоровья: Здоров / Риск).
- **Срок окупаемости (CAC Payback Period):** [N] месяцев.

### 4. Анализ чувствительности (Sensitivity Analysis)
Как изменится LTV при колебании ключевых факторов:
- Если ключевая ставка вырастет на +2% (удорожание фондирования CoF): LTV упадет на [X%].
- Если кредитный риск (PD) вырастет на +1%: LTV упадет на [Y%], экономика сойдется только при росте CAC на [Z%].
```

## Правила

- Запрещай рассчитывать юнит-экономику кредитных продуктов без учета стоимости фондирования (CoF) и кредитного риска (LLP). Показ валового процентного дохода без вычета стоимости привлечения денег и резервов на дефолты — критическая ошибка, создающая иллюзию прибыльности на убыточном портфеле.
- Досрочное погашение (prepayment rate) должно обязательно учитываться при расчете LTV кредита. Если клиенты гасят кредиты быстрее срока договора, банк получает меньше процентных доходов, что снижает LTV.

## Метрики (Fintech / Lending)

### Outcome metric
**risk-adjusted profit, approved good customers, portfolio margin.** Главный результат и ценность.

### Input metrics
**KYC pass rate, approval rate by risk bucket, time-to-decision, utilization.** Управляемые рычаги outcome.

### Guardrails
**NPL 30/60/90, fraud loss, complaints, regulatory breaches, manual review overload.** Что нельзя ухудшить.

### Diagnostic metrics
**vintage analysis, PD/LGD, funnel by segment, false positives/negatives, channel quality.** Где искать причину.

### Instrumentation
**application_id, risk bucket, decision, KYC steps, repayment/vintage data, fraud flags.** Какие данные нужны.

### Decision rules
- Ship / Iterate / Kill
