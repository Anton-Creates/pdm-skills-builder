# Стратегия управления каталогом (catalog-strategy)

Спроектировать и оптимизировать структуру товарного каталога для ритейл-сетей, e-grocery или крупных e-commerce платформ. Скилл помогает продакту выстроить понятную таксономию, настроить правила персонализации витрины, управлять маловостребованным ассортиментом (Long Tail) и контролировать качество контента.

## Процесс

1. **Спроектируй структуру каталога (Taxonomy & Navigation).** Опиши категории, подкатегории, фасетные фильтры (например: бренд, цвет, размер, жирность для молока).
2. **Настрой правила персонализации каталога (Catalog Personalisation).** Каков алгоритм сортировки товаров на полке для конкретного пользователя? (история покупок, сопутствующие товары).
3. **Опиши управление Long Tail (длинным хвостом ассортимента).** Как мерчандайзить товары с низкой частотой покупок, чтобы они находили своего покупателя без перегрузки основного интерфейса (уценка, подборки, умный поиск).
4. **Определи метрики качества каталога (Catalog KPIs).** Search-to-Detail Rate (конверсия из поиска в карточку), Fill Rate характеристик, доля товаров без фото.
5. **Сохрани вывод** в текущей рабочей директории как `catalog-strategy-[название-продукта].md`.

## Формат вывода

```
## Стратегия управления каталогом: [Бренд]

### 1. Архитектура таксономии и Фильтры (Taxonomy & Facets)
- **Иерархия категорий:** [например: Продукты -> Молочные продукты -> Йогурты].
- **Фасетная фильтрация:** [список ключевых фильтров для категории].
- **Полнота атрибутов (Data Completeness):** требование заполнения 100% обязательных характеристик перед публикацией SKU на витрину.

### 2. Персонализация витрины (Catalog Personalisation)
- **Алгоритм сортировки:** [приоритет товаров на основе прошлых покупок клиента в этой категории (репокупка), далее популярное в регионе].
- **Блок рекомендаций:** «С этим товаром покупают» (cross-sell на основе ассоциативных правил).

### 3. Стратегия Long Tail (Управление длинным хвостом)
- **Проблема:** 80% SKU покупаются редко (низкая оборачиваемость).
- **Решение:**
  - Автоматическое выведение Long Tail товаров в поисковую выдачу по низкочастотным запросам.
  - Включение в тематические персонализированные подборки («Полезный завтрак», «Для дачи»).

### 4. Метрики качества каталога (Catalog KPIs)
- **Search Relevance Rate:** доля кликов по первым 5 результатам поиска.
- **Listing Completeness Score:** средний % заполнения карточки товара (фото + описание + свойства). Цель > 95%.
- **Zero Search Results Rate:** % поисковых запросов с нулевым результатом (цель < 1.5%).
```

## Правила

- Запрещай публикацию товаров с незаполненными ключевыми характеристиками или без фотографий. Карточка без фото снижает конверсию в покупку на 80%.
- Обязательно учитывай Long Tail товары. Перегружать ими главную витрину нельзя, но они должны быть легко находимы через качественный текстовый поиск.
- Пиши на русском языке.

## Метрики (E-commerce / Retail)

### Outcome metric
**contribution margin per order, repeat purchase, paid orders.** Главный результат и ценность.

### Input metrics
**checkout conversion, add-to-cart, availability, delivery promise accuracy, promo conversion.** Управляемые рычаги outcome.

### Guardrails
**gross margin after returns, refund rate, return abuse, stockout, payment failure.** Что нельзя ухудшить.

### Diagnostic metrics
**funnel drop-off, category, device, payment method, delivery slot, promo dependency.** Где искать причину.

### Instrumentation
**cart_id, order_id, SKU, inventory, payment events, return/refund reason.** Какие данные нужны.

### Decision rules
- Ship / Iterate / Kill
