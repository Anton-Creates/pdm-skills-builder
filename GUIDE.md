## Как это устроено на разных платформах?

Поскольку продакт-менеджеры и разработчики используют разные инструменты, наши 106 скиллов могут работать в разных парадигмах:

### 1. 💻 В терминале (Claude Code CLI / Pi Agent)
*   **Как это работает:** Вызов **по требованию (On-Demand)**. Скиллы лежат в папке `~/.claude/skills/`.
*   **Как использовать:** В чате терминала вы пишете слэш-команду: `/prd [идея]` или `/saas-metrics [данные]`. Claude Code подгружает инструкции конкретного скилла только в момент вызова команды, экономя токены в остальное время беседы.

### 2. 🔌 Через MCP (Model Context Protocol) в приложениях (Claude Desktop, VS Code)
*   **Как это работает:** Интеграция в виде **активных инструментов (Tools)**. MCP — это открытый стандарт Anthropic для подключения внешних данных к LLM.
*   **Как использовать:** Вы можете запустить локальный MCP-сервер, который будет "отдавать" эти 106 скиллов. Когда вы пишете в Claude Desktop: *«Сделай мне PRD по этой идее»*, модель сама понимает, что у неё есть инструмент `run_prd_skill`, автоматически запускает его в фоне и выдает результат.

### 3. 📝 В редакторах кода (Cursor / VS Code Roo/Cline)
*   **Как это работает:** **Фоновые правила (Background Rules)**. Инструкции компилируются в один файл правил в корне проекта (`.cursorrules` или `.clinerules`).
*   **Как использовать:** ИИ-ассистент постоянно держит эти правила в своей "памяти" (системном промпте). Любой ваш вопрос в чате Cursor автоматически анализируется через призму этих 106 продуктовых скиллов.

### 4. 👥 В командном репозитории (GitHub Copilot / Codex)
*   **Как это работает:** **Командные стандарты (Team Guidelines)**. Инструкции компилируются в файл `.github/copilot-instructions.md`.
*   **Как использовать:** Все разработчики и продакты, работающие в этом репозитории через Copilot, автоматически получают подсказки и генерацию кода, соответствующую вашим продуктовым стандартам (например, структура User Stories или критерии готовности).

### 5. 🤖 В веб-интерфейсах (ChatGPT, Claude.ai, Custom GPTs)
*   **Как это работает:** **Ручной промптинг (Manual Prompting)**. Менеджер собирает выбранные скиллы в файл `SYSTEM_PROMPT.md`.
*   **Как использовать:** Вы просто открываете этот файл, копируете нужный кусок и вставляете его в Custom Instructions в настройках своего профиля ChatGPT или при создании своего кастомного бота (Custom GPT).

## Установка


### Шаг 1: Откройте терминал

Вам нужно приложение терминала. На Mac откройте **Терминал** (найдите его через Spotlight). На Windows используйте **PowerShell** или **Git Bash**.

### Шаг 2: Клонируйте репозиторий

```bash
git clone https://github.com/Tony-Leoni/pdm-skills.git
```

Это скачает все 20 скиллов на ваш компьютер.

### Шаг 3: Скопируйте скиллы в Claude Code

```bash
cp -r pdm-skills/skills/* ~/.claude/skills/
```

Это скопирует каждый скилл в папку, где Claude Code ищет свои скиллы.

> **Хотите скопировать только некоторые?** Скопируйте их по отдельности:
> ```bash
> cp -r pdm-skills/skills/prd ~/.claude/skills/
> cp -r pdm-skills/skills/decision-doc ~/.claude/skills/
> ```

### Шаг 4: Перезапустите Claude Code

Полностью выйдите из Claude Code и запустите его снова. Скиллы загружаются при старте.

### Шаг 5: Проверьте работу

Введите `/` в Claude Code. Вы должны увидеть ваши новые скиллы в списке автодополнения наряду с теми, что у вас уже были.

**Если вы их не видите**, проверьте:
- Существует ли папка `~/.claude/skills/` и содержит ли она подпапки (по одной на каждый скилл).
- Содержит ли каждая подпапка файл `SKILL.md`.
- Перезапустили ли вы Claude Code после копирования.

## Как использовать скилл

Каждый скилл работает по одному шаблону:

```
/имя-команды [ваши вводные данные]
```

Наберите слэш-команду, добавьте контекст, нажмите Enter. Claude сделает всё остальное.

### Ваш первый скилл — попробуйте прямо сейчас

Скопируйте и вставьте это в Claude Code:

```
/decision-doc Стоит ли нам делать свою систему авторизации или использовать Auth0?
```

Вы получите структурированный документ о решении с вариантами, компромиссами, рекомендацией и оценкой обратимости — примерно за 30 секунд.

### Советы по получению лучших результатов

1. **Будьте конкретны.** `/prd Инструмент для PM` выдаст вам слишком общий PRD. `/prd Slack-бот, который собирает сводку непрочитанных каналов каждое утро и выделяет задачи` выдаст то, с чем уже можно работать.
2. **Добавляйте контекст.** Большинство скиллов работают лучше, когда вы задаете ограничения. `/ab-test-design Перенос кнопки CTA на липкую шапку. Текущая конверсия 3.2%, 800 визитов в день` даёт Claude цифры, необходимые для расчёта выборки.
3. **Вставляйте сырые данные.** Ряд скиллов принимает хаотичный текст и структурирует его за вас. `/stakeholder-update` работает лучше всего, если вы просто вставите ваши сумбурные заметки. `/feedback-analyzer` отлично переваривает скопированные отзывы или путь к файлу CSV.
4. **Объединяйте скиллы в цепочки.** Сначала запустите `/persona`, затем скормите результат в `/prd`, а после — в `/user-stories`. Скиллы спроектированы так, чтобы дополнять друг друга.

## Пояснительная бригада по сборкам (Рекомендуемые комбо-пресеты)

Поскольку менеджер `manager.py` устанавливает пресеты **накопительно** (не удаляя старые, пока вы не напишете `clean`), вы можете собирать индивидуальные рабочие среды под разные типы продуктов.

Вот 5 готовых «рецептов сборки» под конкретные роли:

### 1. 🛠️ Универсальный Продакт-Крафт (Для любого PM общего профиля)
Подходит для классического цикла: исследования -> проектирование -> гипотезы.
*   **Сборка:**
    ```bash
    python manager.py preset core
    python manager.py preset discovery
    python manager.py preset craft
    ```
*   **Что дает:** Полный контроль над созданием PRD, User Stories, картами CJM, проведением интервью и построением деревьев гипотез/метрик.

### 2. 📈 B2B / SaaS Продакт (Продукты по подписке)
Ориентирован на когортный анализ, тарифные сетки и юнит-экономику подписок.
*   **Сборка:**
    ```bash
    python manager.py preset core
    python manager.py preset saas
    python manager.py preset craft
    ```
*   **Что дает:** Расчет MRR/NRR, проектирование Value Metric, дизайн A/B тестов цен, анализ защитных барьеров (Moat) и выкатка фич.

### 3. 🛒 Marketplace & E-commerce (Маркетплейсы, доставка, ритейл)
Для работы над ростом конверсии, балансом ликвидности продавцов и покупателей, логистикой и качеством каталога.
*   **Сборка:**
    ```bash
    python manager.py preset core
    python manager.py preset marketplace
    python manager.py preset ecom-retail
    ```
*   **Что дает:** Аудит воронки чекаута, управление каталогом, выбор моделей FBO/FBS/DBS, оптимизация дарксторов и борьба с фродом.

### 4. 🏦 Fintech & Banking (Кредитные и транзакционные продукты)
Для работы в жестком регуляторном комплаенсе с фокусом на расчет рисков и юнит-экономики кредитных продуктов.
*   **Сборка:**
    ```bash
    python manager.py preset core
    python manager.py preset fintech
    python manager.py preset craft
    ```
*   **Что дает:** Аудит 115-ФЗ комплаенса, расчет кредитной юнит-экономики (CoF, LLP, PD), построение дерева метрик и роадмапов.

### 5. 🏢 Enterprise & B2G (Внутренние ИТ-системы и Гос-сервисы)
Для автоматизации внутренних процессов компании, работы со стейкхолдерами и подготовки тендерной документации.
*   **Сборка:**
    ```bash
    python manager.py preset core
    python manager.py preset enterprise-gov
    ```
*   **Что дает:** Заполнение ТЗ (RFP), запуск софта по филиалам (Enterprise Rollout), проектирование госуслуг по ФЗ-210 и интеграцию по СМЭВ.

## Решение проблем (Troubleshooting)

### «Я пишу /имя-команды, но ничего не происходит»

- Убедитесь, что перезапустили Claude Code после копирования файлов.
- Проверьте, существует ли файл `~/.claude/skills/[имя-команды]/SKILL.md`.
- Имя команды должно в точности совпадать с именем папки.

### «Вывод получается слишком банальным/общим»

Ваш ввод, скорее всего, слишком размыт. Сравните:
- Плохо: `/prd Инструмент продуктивности`
- Хорошо: `/prd Slack-бот, который отправляет ежедневную сводку непрочитанных каналов каждому пользователю, выделяя сообщения, в которых его упомянули или где на него назначена задача`

Больше контекста = лучше результат. Добавляйте ограничения, цифры, целевую аудиторию и цели.

### «Скилл задаёт мне вопросы вместо генерации вывода»

Некоторые скиллы пресекают слишком бедные вводные данные. Это сделано намеренно — размытый PRD хуже, чем его отсутствие. Ответьте на вопросы, и скилл продолжит генерацию.

### «Мне нужны только некоторые скиллы, а не все 20»

Скопируйте только нужные вам папки:
```bash
cp -r claude-code-pm-skills/skills/prd ~/.claude/skills/
cp -r claude-code-pm-skills/skills/prioritize ~/.claude/skills/
```

Каждый скилл автономен. Никакой скилл не зависит от другого.

### «Я хочу изменить логику скилла»

Отредактируйте файл `SKILL.md` напрямую в папке скилла. Это обычный markdown-файл с инструкциями. Меняйте процесс, формат вывода, правила — всё, что вам нужно. Ваши изменения сохранятся.

## Часто задаваемые вопросы (FAQ)

**Нужен ли мне API-ключ?**
Нет. Скиллы запускаются внутри Claude Code и используют вашу существующую подписку.

**Работают ли скиллы без интернета (offline)?**
Большинство скиллов работают офлайн. Исключения: `/product-teardown`, `/competitor-scan` и `/market-sizing` используют веб-поиск для сбора данных, поэтому им нужно интернет-соединение.

**Могу ли я поделиться настроенным скиллом с командой?**
Да. Скилл — это просто папка с файлом SKILL.md. Поделитесь этой папкой, вашим коллегам достаточно положить её в свою директорию `~/.claude/skills/`.

**Отправляют ли скиллы мои данные на сторонние серверы?**
Нет. Скиллы — это локальные инструкции. Ваши данные остаются внутри вашей сессии Claude Code.

**Можно ли использовать эти скиллы в веб-версии Claude (claude.ai)?**
Нет. Скиллы — это фича Claude Code (консольного CLI-инструмента). На claude.ai они работать не будут.

**Как обновлять скиллы при выходе новых версий?**
Подтяните последние изменения из репозитория и скопируйте файлы заново:
```bash
cd claude-code-pm-skills
git pull
cp -r skills/* ~/.claude/skills/
```

**Могу ли я создавать свои скиллы?**
Конечно. Создайте папку в `~/.claude/skills/`, добавьте файл `SKILL.md` в таком же формате, как у существующих скиллов, и перезапустите Claude Code. Шаблон можно подсмотреть в любом скилле этого репозитория.
