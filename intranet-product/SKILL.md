# Корпоративный интранет (intranet-product)

Спроектировать архитектуру и сервисы корпоративного интранет-портала для сотрудников компании (поиск контактов, заказ справок, внутренние новости).

## Процесс
1. **Спроектируй глобальный поиск (Enterprise Search).** Поиск коллег по имени, департаменту, навыкам или внутренним документам.
2. **Опиши структуру сервисов самообслуживания.** Заказ справок 2-НДФЛ, согласование отпусков, бронирование переговорных.
3. **Определи метрики.** DAU/MAU портала, удовлетворенность сервисами (CSAT).

## Формат вывода
```
## Спецификация интранет-портала: [Название]
- **Глобальный поиск:** интеграция с Active Directory (AD) для автоматического обновления базы контактов.
- **Сервис самообслуживания:** автоматизированная отправка заявки на отпуск в 1С:ЗУП.
```

## Метрики

### Универсальное правило метрики
Если вы предлагаете метрику, ответьте на 5 вопросов:
1. **Кто владеет этой метрикой?**
2. **Как часто её смотрим?**
3. **Какие события её считают?**
4. **Какой порог решения?**
5. **Как её можно испортить или накрутить?**
