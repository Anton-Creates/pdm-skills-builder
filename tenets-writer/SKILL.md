# Product Tenets Writer

Сформулировать не лозунги, а практические принципы, по которым команда будет принимать сложные решения.

## Формат вывода

```md
## Product Tenets: [Продукт]

| Принцип | Почему критично | Где риск нарушить | Как проверять | Guardrail |
|---|---|---|---|---|
| | | | | |
```

## Правила

- Tenet должен помогать выбирать между двумя хорошими вариантами.
- У каждого принципа должна быть проверка и guardrail.
- Не пиши абстракции вроде “клиент в центре”, если непонятно, какое решение это меняет.
- Пиши на русском языке.

## Метрики

### Универсальное правило метрики
Если вы предлагаете метрику, ответьте на 5 вопросов:
1. **Кто владеет этой метрикой?**
2. **Как часто её смотрим?**
3. **Какие события её считают?**
4. **Какой порог решения?**
5. **Как её можно испортить или накрутить?**
