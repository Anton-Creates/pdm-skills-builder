# Claude Code PdM Skills (на русском)

106 готовых к работе скиллов для продакт-менеджеров (PdM), использующих Claude Code.

Никакой воды. Никаких шаблонных промптов. Реальные рабочие процессы PM, которые вы можете запустить одной командой `/имя-команды`.

## Что внутри

### Discovery и стратегия (Исследования и стратегия)
| Скилл | Что делает |
|---|---|
| `/product-teardown` | Разбор любого продукта: бизнес-модель, UX-решения, петли роста, защитные барьеры |
| `/competitor-scan` | Анализ конкурентного ландшафта с картой позиционирования и поиском свободных ниш |
| `/user-interview-prep` | Бриф исследования с гипотезами, гайдом интервью и вопросами по «Маминому тесту» |
| `/feedback-analyzer` | Классификация сырого фидбека (CSV или текст) по темам, тональности, срочности + паттерны |
| `/market-sizing` | Оценка TAM/SAM/Some с методологией сверху вниз + снизу вверх |
| `/persona` | Поведенческие персоны пользователей, влияющие на продуктовые решения — без маркетинговой воды |

## Установка

### Вариант 1: Копирование отдельных скиллов

```bash
# Клонировать репозиторий
git clone https://github.com/Tony-Leoni/pdm-skills.git

# Скопировать нужные скиллы в папку скиллов Claude Code
cp -r pdm-skills/skills/prd ~/.claude/skills/
cp -r pdm-skills/skills/decision-doc ~/.claude/skills/
# ... и так далее
```

### Вариант 2: Копировать все скиллы

```bash
git clone https://github.com/Tony-Leoni/pdm-skills.git
cp -r pdm-skills/skills/* ~/.claude/skills/
```

### Вариант 3: Установка через плагин (скоро)

```bash
/plugin marketplace add Tony-Leoni/pdm-skills
/plugin install pdm-skills
```

После копирования перезапустите Claude Code. Введите `/`, чтобы увидеть ваши новые скиллы в списке автодополнения.

## Использование

Каждый скилл запускается с помощью слэш-команды `/имя-команды`:

```
/prd Утилита, которая помогает PM приоритизировать фичи с помощью AI-оценки

/product-teardown Notion

/ab-test-design Перенос кнопки CTA выше первого экрана на странице тарифов

/feedback-analyzer /path/to/customer-feedback.csv

/decision-doc Стоит ли нам делать свою систему авторизации или использовать Auth0?
```

Скиллы умеют автоматически определять контекст. Если вы вставите CSV и попросите Claude проанализировать отзывы, он может запустить команду `/feedback-analyzer` автоматически.

## Принципы проектирования

- **Своё мнение вместо общих фраз (Opinionated).** Каждый скилл пресекает размытые формулировки и заставляет быть конкретным.
- **Максимум 1-2 страницы.** Плотность важнее длины. Никаких 20-страничных PRD.
- **Процесс вместо промптов.** Это многошаговые процессы со структурированным результатом, а не просто запросы, которые вы могли бы написать сами.
- **Plug and play.** Никаких зависимостей. Никаких API-ключей. Скопируйте папку и пользуйтесь.
