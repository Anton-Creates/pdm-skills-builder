# Создание структуры питч-дека (presentation-design)

Спроектировать убедительную раскадровку презентации (Pitch Deck, Discovery Defense) для защиты продуктовой концепции перед инвесторами, C-level или продуктовым комитетом. Скилл строит 10-слайдовый фреймворк с говорящими заголовками (Action Titles), рекомендациями по визуализации и конкретными блоками контента — без воды и общих фраз.

## Процесс

1. **Уточни вводные данные:**
   - **Цель защиты:** Go/No-go по Discovery, запрос ресурсов, получение финансирования.
   - **Аудитория:** Инвесторы / C-level / Продуктовый комитет / Внешняя комиссия.
   - **Ключевая мысль (Key Message):** Одна главная идея, которую аудитория должна вынести (например: «Проблема валидирована, экономика сходится, нужен пилот»).

2. **Применяй 10-слайдовый Gold Standard фреймворк:**
   - **Слайд 1: Титульный.** Название, суть в 1 предложении, команда, статус (Go / Pilot / Research).
   - **Слайд 2: Паспорт концепции (Executive Summary).** Тип проекта, аудитория, проблема, монетизация, ключевое преимущество, параметры пилота — всё в одной таблице.
   - **Слайд 3: Проблема и Аудитория.** Разрыв в пользовательском опыте, JTBD, % подтверждения проблемы из количественных опросов.
   - **Слайд 4: Рынок и Альтернативы.** TAM / SAM / SOM, средний чек, матрица конкурентов и почему они не закрывают проблему.
   - **Слайд 5: Решение и User Journey.** Наглядная воронка или Checkout-схема, принцип работы продукта, ключевые экраны.
   - **Слайд 6: Валидация.** Результаты качественных и количественных тестов, итерации продукта после обратной связи.
   - **Слайд 7: MVP и Пилот.** Release 1 (что делаем сейчас) vs Release 2+ (бэклог), конкретные KPI пилота (конверсия, кол-во сделок, NPS/CSI).
   - **Слайд 8: Экономика.** Unit-экономика (ARPU, COGS, Contribution Margin, CAC), BEP, прогноз P&L на 12 месяцев.
   - **Слайд 9: GTM-стратегия.** Каналы с минимальным CAC, виральные механики, compliance-ограничения и лицензии.
   - **Слайд 10: Итоги и Рекомендация.** Выводы по 4 блокам: Проблема ✓ → Решение ✓ → Экономика ✓ → GTM ✓ → **Запустить пилот**.

3. **Для каждого слайда сформируй:**
   - **Action Title** — говорящий заголовок с выводом, а не темой («Экономика сходится» вместо «Финансы»).
   - **Визуализация** — что именно изображено (воронка, таблица, P&L-диаграмма, 3 карточки).
   - **Контент** — 3-5 конкретных буллитов с цифрами, без воды.

4. **Sanity Check:**
   - Прочитай только Action Titles подряд — история должна быть понятна без остального текста.
   - На последнем слайде обязателен чёткий призыв к действию (Go / No-go / Пилот).

## Формат вывода

```
## Слайд [N]: [Action Title]
- **Визуализация:** [тип визуала]
- **Контент:**
  - [факт / цифра / тезис 1]
  - [факт / цифра / тезис 2]
  - [факт / цифра / тезис 3]
```

## Правила и ограничения
- Максимум 3-5 сущностей на слайд — стена текста недопустима.
- Все данные должны быть конкретными: цифры из опросов, расчёты конверсий, объём рынка в рублях/штуках.
- Запрещены расплывчатые формулировки типа «мы улучшим опыт» — только «конверсия в регистрацию ≥ 15%».
- Структура рассчитана на доклад 15-20 минут + Q&A.

## Метрики успеха скилла
- Аудитория понимает суть, прочитав только Action Titles (Headline Test).
- Последний слайд содержит явную рекомендацию с конкретным следующим шагом.
- В слайде с экономикой есть BEP и прогнозный P&L.
